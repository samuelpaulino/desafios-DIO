Comunicação Cliente X Server

Voltar ao início

Para que possamos estabelecer a relação de Client e do Server, é necessário definir o que cada um é nesse conjunto. Entende-se por "Client" o usuário final, que faz a solicitação através de um navegador para que possa acessar tal conteúdo. Aquelas informações que o Client visualiza, ou seja, o que lhe é apresentado, também podemos considerar como sendo o "front-end". Já o "Server", como seu nome já diz, é a parte que contém as informações codificadas e que respondem ao Client quando este faz suas requisições. Server está envolvido com a parte conhecida como "back-end" da Programação. Por meio do que é chamada de rede internet, o Client faz uma requisição utilizando um protocolo (HTTP), em que a informação é transmitida de seu computador ou smartphone, passa pelo modem que também é interligado em uma rede mundial (internet) e através dessa rede a requisição chega ao servidor que volta a informação solicitada do client para ele.

O que é HTML

Voltar ao início

Por definição, HTML (HiperText Markup Language) significa Linguagem de Marcação de Hipertexto. É o que permite inserir conteúdo e estabelece a estrutura básica de um site, ou seja, para que um navegador interprete um documento ele deve estar na extensão .html. A estrutura do HTML é composta por três TAG's principais, que são elas: HTML, HEAD e BODY. Todo arquivo a ser feito em HTML deve inciar e terminar com a TAG html. Já dentro da TAG html, deve-se incluir as TAGs head e body. A TAG head consiste em tudo aquilo que será carregado antes de o usuário viualizar a página. Já a TAG body consiste naquilo que o usuário visualizará.

TAGs, atributos e listas

Voltar ao início

Como já iniciado a explicação no item acima, TAGs são parte da estrutura básica da formatação em HTML. As TAGs aprendidas no curso "Formação HTML Web Developer" da DIO até aqui tem basicamente a mesma estrutra: iniciam-se com um "sinal de menor" como abertura de tal TAG, a sigla da TAG, "sinal de maior", seguida de um conteúdo e depois fecha-se a TAG com "sinal de menor", barra, sigla da TAG e por fim "sinal de maior". Já a respeito de atributos, posso resumir como sendo algumas ferramentas presentes dentro de TAGs que servem, por exemplo, para a formatação daquilo que é apresentado ao Client. Também aprendi a respeito de listas ordenadas e não-ordenadas. Listas ordenadas, como o próprio nome já diz, respeitam uma ordem e são automaticamente enumeradas quando estamos formatando em .html. Já as listas não-ordenadas são aquelas que não precisam respeitar uma ordem crescente como uma lista enumerada. Um exemplo desse tipo de lista é a lista do índice desta página.
